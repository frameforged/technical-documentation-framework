# Structure Rules

## Standard documentation package

Use this package for broad technical product documentation:

```text
docs/
├── README.md
├── overview.md
├── getting-started.md
├── concepts.md
├── architecture.md
├── configuration.md
├── api-reference.md
├── sdk-guide.md
├── integration-guide.md
├── examples.md
├── troubleshooting.md
├── glossary.md
└── faq.md
```

Not every product needs every file. Choose the files based on the product type and user request.
