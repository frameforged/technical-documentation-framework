# Example Prompt — AI Agent Orchestration Platform

Use the Technical Documentation Framework to create documentation for an AI agent orchestration platform.

## Product description

The platform allows teams to create, configure, run, observe, and govern AI agents in an enterprise environment.

## Capabilities

- Agent creation
- Tool registration
- Skill management
- Short-term memory
- Long-term memory
- Guardrail configuration
- Runtime orchestration
- Trace and observability
- Role-based access control
- Human approval for sensitive actions

## Target audience

- Developers
- Solution architects
- Platform engineers
- Security reviewers

## Documentation requirements

- Write in the language of this conversation unless explicitly overridden.
- Include a detailed glossary.
- Explain terms such as Agent, Tool, Skill, Brain, Memory, Short-term Memory, Long-term Memory, Guardrail, Runtime, Trace, and Orchestration.
- Include realistic enterprise examples.
- Include incorrect usage examples.
- Include edge cases.
- Include troubleshooting guidance.
- Clearly state assumptions and product boundaries.

## Expected output

Create a Markdown documentation package with:

- README
- Overview
- Getting Started
- Concepts
- Architecture
- Configuration
- Examples
- Troubleshooting
- Glossary
- FAQ

## Delivery format (optional variations)

The same prompt can end with a delivery instruction, and Phase 10 handles the rest:

> Deliver the final documentation as a single Word document with a title page,
> table of contents, and page numbers.

> Export the package as one self-contained HTML file for our internal wiki.

> Produce a PDF, and include screenshots of the admin console in the
> configuration section.
