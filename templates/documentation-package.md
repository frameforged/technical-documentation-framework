# Documentation Package Template

Use this template when producing a complete documentation package.

```text
docs/
├── README.md
├── overview.md
├── getting-started.md
├── concepts.md
├── architecture.md
├── configuration.md
├── api-reference.md
├── examples.md
├── troubleshooting.md
├── glossary.md
└── faq.md
```

## README.md

Purpose: Short entry point for the product.

Must include:

- Product description
- What it does
- Who it is for
- Quick start
- Documentation map
- Example usage

## overview.md

Purpose: Explain the product and problem at a high level.

Must include:

- Product summary
- Problem statement
- Main capabilities
- Target users
- Product boundaries

## getting-started.md

Purpose: Help the reader complete the first successful use.

Must include:

- Prerequisites
- Installation or setup
- First configuration
- First successful execution
- Next steps

## concepts.md

Purpose: Explain the conceptual model.

Must include:

- Core concepts
- How concepts relate
- Common misunderstandings
- Links to glossary

## architecture.md

Purpose: Explain system structure and behavior.

Must include:

- System context
- Components
- Runtime flow
- Data flow
- Integration points
- Security model
- Observability

## configuration.md

Purpose: Explain configuration behavior.

Must include:

- Configuration options
- Defaults
- Required values
- Optional values
- Environment-specific notes
- Common mistakes

## api-reference.md

Purpose: Document API behavior.

Must include:

- Authentication
- Common headers
- Error model
- Endpoints
- Request examples
- Response examples

## examples.md

Purpose: Show practical usage.

Must include:

- Simple example
- Realistic example
- Incorrect usage example
- Edge-case example

## troubleshooting.md

Purpose: Help diagnose and fix common issues.

Must include:

- Symptoms
- Possible causes
- Checks
- Fixes
- Prevention

## glossary.md

Purpose: Define terminology.

Must include:

- Short definition
- Detailed explanation
- Product-specific meaning
- Example
- Common confusion
- Related terms

## faq.md

Purpose: Answer recurring questions.

Must include:

- Conceptual questions
- Usage questions
- Integration questions
- Operational questions
